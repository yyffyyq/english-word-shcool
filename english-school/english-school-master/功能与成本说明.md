## 一、项目概述

本项目是一款参考「百词斩」学习体验的校园背单词微信小程序，采用 **班级制** 管理模式：

- **学生**通过小程序背单词，学习内容由所属班级的教师指定
- **教师**通过小程序管理班级、布置词书与学习任务
- **管理员**通过 Web 管理端审批教师、管理账号

**核心特点：**

- 免费使用，不涉及支付
- 个人主体即可注册，无需企业认证
- 教师注册需管理员审批，学生注册自动通过
- 教师端与学生端导航栏内容不同

## 二、功能说明

### 2.1 系统角色

| 角色 | 使用入口 | 注册/认证方式 |
|------|----------|---------------|
| 超级管理员 | Web 管理端 | 预置账号，账号密码登录 |
| 教师 | 微信小程序 | 微信登录 → 填写信息 → **待审批** → 管理员通过后可用 |
| 学生 | 微信小程序 | 微信登录 → 填写信息 → **自动通过** → 选择班级后加入 |

**角色关系：**

```
管理员 ──审批──▶ 教师 ──创建──▶ 班级 ◀──加入── 学生（多个）
                              │
                              └── 布置词书/学习任务
```

## 三、服务器配置建议

### 3.1 推荐配置（1 校内测）

| 项目 | 配置 | 说明 |
|------|------|------|
| CPU | 2 核 | 足够 |
| 内存 | 2 GiB | 内测够用，需控制 JVM 内存 |
| 硬盘 | 40 GiB SSD | 足够 |
| 带宽 | 3 Mbps 固定 | 足够 |
| 操作系统 | Alibaba Cloud Linux 3 | 推荐 |
| 地域 | 大陆节点 | 需 ICP 备案 |

### 3.2 部署架构

```
小程序 / Web 管理端
        │
        ▼
   Nginx (HTTPS)
        │
        ▼
  Spring Boot 后端
   ├── MySQL（数据库）
   └── Redis（缓存）
        │
        ▼
   阿里云 OSS（单词图片/音频）
```

### 3.3 资源需求评估

| 资源 | 需求程度 | 说明 |
|------|----------|------|
| 带宽 | 低 | API 请求为主，媒体文件走 OSS |
| 内存 | 低~中 | 2G 内测够用，JVM 限制 512M |
| CPU | 低 | 以 CRUD 为主，无复杂计算 |
| 硬盘 | 低 | 词书数据量小，40G 足够 |

**计费项：**

| 计费项 | 说明 | 内测费用 |
|--------|------|----------|
| 存储费 | 图片/音频占用的空间 | 约 ¥0.1～0.3/月 |
| 流量费 | 学生下载图片/音频 | 约 ¥0.5～1/月 |
| 请求费 | 每次访问文件 | 约 ¥0.1/月 |

**内测场景估算（50～100 学生）：**

```
词书存储：约 1～2 GB → ¥0.1～0.3/月
学生流量：约 200MB～1GB/天 → ¥0.5～1/月
────────────────────────────────
合计：约 ¥1～2/月 → 一年 ¥10～20
```

> OSS 需要花钱，但在总成本中占比很小（约 5%～10%），且能节省服务器带宽和硬盘空间，正式使用时建议使用。
